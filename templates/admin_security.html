IP 访问控制

配置 IP 白名单、黑名单以及共享密钥等高级网络层限制。

IP 白名单

启用后，只允许白名单中的 IP 访问系统。支持单个 IP、CIDR 和 IP 范围。

IP 黑名单

启用后，黑名单中的 IP 将被拒绝访问。黑名单优先级高于白名单。

共享密钥验证

要求请求头包含正确的共享密钥，适用于内部服务调用场景。

支持格式：单个 IP (192.168.1.100)、CIDR (192.168.1.0/24)、IP 范围 (192.168.1.100-192.168.1.200)。以 # 开头的行为注释。
格式同白名单。黑名单优先级高于白名单，即使 IP 在白名单中，如果也在黑名单中则会被拒绝。
用于验证请求的共享密钥，建议使用强随机字符串。
请求头中携带共享密钥的字段名称。

IP 访问控制说明

配置 Nginx 反向代理时的注意事项和配置示例。

工作原理 IP 访问控制在应用层生效，通过读取 X-Forwarded-For 请求头获取真实客户端 IP。如果使用 Nginx 反向代理，需要正确配置 proxy_set_header。
Nginx 配置示例 
location / {
    proxy_pass http://127.0.0.1:5000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_set_header X-Internal-Secret "your-secret-here";
}
紧急恢复机制 如果配置错误导致无法访问，可以使用命令行参数快速恢复：
python app.py --disable-ip-control
测试建议 启用白名单前，建议先在黑名单模式下测试，确保配置正确。启用后，可以使用无痕模式或其他设备测试是否正常工作。
