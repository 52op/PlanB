# 登录频率限制说明

## 功能概述

为了防止暴力破解和频繁请求攻击，系统对以下登录方式实施了频率限制：

1. **全局密码访问**（访客访问）
2. **用户密码登录**（账号密码登录）

邮箱验证码登录和注册不受此限制，因为验证码本身已提供足够的保护。

## 限制策略

### 渐进式惩罚机制

系统采用基于 IP 地址的渐进式惩罚策略：

| 失败次数 | 等待时间 | 说明 |
|---------|---------|------|
| 1-3 次 | 无限制 | 允许立即重试 |
| 3-5 次 | 30 秒 | 短时间冷却 |
| 5-10 次 | 5 分钟 | 中等惩罚 |
| 10 次以上 | 30 分钟 | 严重惩罚 |

### 重置机制

- **登录成功**：立即清除该 IP 的所有失败记录
- **自动清理**：2小时无活动的记录会被自动清理（节省内存）

## 技术实现

### 核心组件

- **服务模块**：`services/rate_limit.py`
- **存储方式**：内存字典（可扩展为 Redis）
- **线程安全**：使用 `threading.Lock` 保证并发安全

### 主要函数

```python
# 检查是否被限制
check_rate_limit(ip_address) -> (is_allowed, wait_seconds, failure_count)

# 记录失败
record_login_failure(ip_address) -> (wait_seconds, failure_count)

# 记录成功（清除记录）
record_login_success(ip_address)

# 获取客户端真实 IP
get_client_ip(request) -> ip_address

# 格式化等待时间
format_wait_time(seconds) -> "X 分钟" 或 "X 小时"
```

### IP 获取优先级

系统会按以下优先级获取客户端真实 IP：

1. `X-Forwarded-For` 头（取第一个 IP）
2. `X-Real-IP` 头
3. `CF-Connecting-IP` 头（Cloudflare）
4. `request.remote_addr`（直连）

## 用户体验

### 错误提示

- **前 3 次失败**：`用户名或密码错误（已失败 X 次）`
- **触发限制后**：`用户名或密码错误。连续失败 X 次，请在 Y 后重试`

### 前端保护

- 提交按钮在提交后 3 秒内禁用，防止重复点击
- 显示"提交中..."状态提示

## 配置参数

可在 `services/rate_limit.py` 中调整：

```python
MAX_ATTEMPTS_BEFORE_DELAY = 3  # 开始限制的失败次数

DELAY_LEVELS = [
    (3, 30),      # 3-5次：30秒
    (5, 300),     # 5-10次：5分钟
    (10, 1800),   # 10次以上：30分钟
]
```

### 优势

- 多进程/多服务器共享数据
- 持久化存储
- 自动过期（TTL）
- 更好的性能

### 监控建议

1. 记录被限制的 IP 地址到日志
2. 统计每日被限制的次数
3. 设置告警阈值（如单个 IP 超过 20 次失败）
4. 考虑集成 IP 黑名单服务

## 安全注意事项

1. **代理环境**：确保正确配置代理头信任
2. **NAT 网络**：同一 NAT 后的多个用户共享 IP，可能误伤
3. **动态 IP**：攻击者可能通过更换 IP 绕过限制
4. **分布式攻击**：需要配合其他防护措施（如 WAF、验证码）
