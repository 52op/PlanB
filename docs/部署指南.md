# 部署说明

## 性能优化

### 问题
Flask 默认的开发服务器（`app.run()`）性能极差：
- 单线程，一次只能处理一个请求
- 页面加载慢，多用户访问时会排队
- 不适合生产环境

### 解决方案

项目已配置自动切换服务器：

#### 开发模式（debug: true）
使用 Flask 开发服务器，支持热重载

#### 生产模式（debug: false）
自动使用 **Waitress** 高性能服务器：
- 多线程并发处理
- 性能提升 10-50 倍
- 适合 Windows 生产环境

## 快速启动

### Windows

```bash
# 方式1：使用启动脚本（推荐）
start.bat

# 方式2：手动启动
venv\Scripts\activate
pip install -r requirements.txt
python app.py

# 方式3：首次部署或数据库结构变更后
python app.py --update-db  # 自动同步数据库结构
```

### Linux/Mac

```bash
# 安装依赖
pip install -r requirements.txt

# 生产环境推荐使用 Gunicorn
pip install gunicorn

# 启动（4个工作进程）
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()[0]"
```

## 配置文件

编辑 `config.yaml`：

```yaml
# 端口
port: 5000

# 调试模式（生产环境设为 false）
debug: false

# 数据库路径
database_path: sqlite:///app.db

# 密钥（生产环境必须配置）
secret_key: "your-secret-key-here"
```

## 性能对比

| 服务器 | 并发能力 | 响应时间 | 适用场景 |
|--------|---------|---------|---------|
| Flask 开发服务器 | 1 请求/次 | 慢 | 仅开发 |
| Waitress | 多线程 | 快 | Windows 生产 |
| Gunicorn | 多进程 | 最快 | Linux 生产 |

## 反向代理配置示例

### Nginx

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static {
        alias /path/to/planning/static;
        expires 30d;
    }
}
```

### Caddy（自动 HTTPS）

```
your-domain.com {
    reverse_proxy localhost:5000
}
```

## 故障排查

### 页面加载慢
1. 检查是否使用了 Flask 开发服务器
2. 确认 `config.yaml` 中 `debug: false`
3. 验证 waitress 已安装：`pip list | grep waitress`

### 图标不显示
1. 检查网络连接（CDN 资源）
2. 清除浏览器缓存
3. 查看浏览器控制台错误

### 数据库锁定
SQLite 在高并发下可能出现锁定，考虑升级到 PostgreSQL 或 MySQL。
