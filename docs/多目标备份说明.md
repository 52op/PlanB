# 多目标备份功能

## 功能概述

多目标备份功能允许您同时将备份文件上传到多个远程存储，提供更好的数据冗余保护。

## 支持的存储类型

- **FTP 服务器**：通过FTP协议上传到远程服务器
- **邮件附件**：将备份文件作为邮件附件发送
- **S3 存储**：上传到AWS S3或兼容的对象存储

## 使用方法

### 1. 配置多个存储目标

1. 进入"管理后台" → "备份管理" → "备份配置"
2. 在"存储方式"部分，可以勾选多个存储选项（如同时勾选"FTP 服务器"和"邮件附件"）
3. 填写每个选中存储类型的配置信息
4. 点击"保存配置"

### 2. 测试连接

配置完成后，建议点击"测试连接"按钮验证所有存储配置是否正确。

### 3. 备份行为

- 执行备份时，系统会依次上传到所有选中的存储目标
- 如果某个目标上传失败，不会影响其他目标的上传
- 备份任务状态：
  - 全部成功：显示"成功"
  - 部分成功：显示"成功"，但会在详情中注明失败的目标
  - 全部失败：显示"失败"

## 优势

1. **数据冗余**：多个备份副本分散在不同存储，降低数据丢失风险
2. **灵活恢复**：可以从任意一个存储恢复数据
3. **成本优化**：可以组合使用免费邮箱和付费云存储

## 注意事项

1. **存储成本**：多目标备份会占用更多存储空间，请注意成本
2. **备份时间**：上传到多个目标会增加备份时间
3. **网络带宽**：确保网络带宽足够支持多目标上传
4. **邮箱限制**：邮件附件通常有大小限制（如25MB），大文件可能发送失败

## 数据库变更

如果从旧版本升级，需要执行数据库迁移：

```bash
# 使用MySQL客户端执行迁移脚本
mysql -u your_username -p your_database < migrations/update_storage_type_field.sql
```

或者在Python中执行：

```python
from app import app, db

with app.app_context():
    # 更新字段长度
    db.engine.execute("ALTER TABLE backup_configs MODIFY COLUMN storage_type VARCHAR(200) NOT NULL")
    db.engine.execute("ALTER TABLE backup_jobs MODIFY COLUMN storage_type VARCHAR(200)")
```

## 兼容性

- 旧的单一存储类型配置仍然兼容
- 系统会自动识别并处理单一存储类型和多存储类型
- 无需修改现有备份记录

## 示例配置

### 场景1：本地FTP + 云端S3

适合需要本地快速恢复和云端长期保存的场景。

- 勾选：FTP 服务器 + S3 存储
- FTP用于快速恢复（局域网速度快）
- S3用于异地容灾（防止本地灾难）

### 场景2：FTP + 邮件

适合小型网站的经济型备份方案。

- 勾选：FTP 服务器 + 邮件附件
- FTP作为主要存储
- 邮件作为额外保险（免费）

### 场景3：三重保护

适合关键业务数据的最高级别保护。

- 勾选：FTP 服务器 + S3 存储 + 邮件附件
- 三个独立的备份副本
- 最大程度降低数据丢失风险
