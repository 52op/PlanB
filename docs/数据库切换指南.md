# 数据库切换指南

本文档说明如何将项目从 SQLite 无损迁移到其他数据库系统。

## 🗄️ 支持的数据库

由于项目使用 SQLAlchemy ORM，理论上支持所有 SQLAlchemy 兼容的数据库：

| 数据库 | 推荐场景 | 并发性能 | 部署难度 | 备份方式 |
|--------|---------|---------|---------|---------|
| **SQLite** | 小型项目、单机部署 | 低 | ⭐ 极简 | 文件复制 |
| **PostgreSQL** | 生产环境、高并发 | 高 | ⭐⭐⭐ 中等 | pg_dump |
| **MySQL/MariaDB** | 通用生产环境 | 高 | ⭐⭐⭐ 中等 | mysqldump |
| **SQL Server** | Windows 企业环境 | 高 | ⭐⭐⭐⭐ 复杂 | SSMS 备份 |
| **Oracle** | 大型企业 | 极高 | ⭐⭐⭐⭐⭐ 很复杂 | RMAN |

## 🎯 推荐选择

### 小型项目（< 1000 用户）
✅ **SQLite**（默认）
- 零配置，开箱即用
- 单文件，易于备份
- 适合个人博客、文档系统

### 中型项目（1000-10000 用户）
✅ **PostgreSQL** 或 **MySQL/MariaDB**
- 更好的并发性能
- 支持多连接
- 适合团队协作、中小企业

### 大型项目（> 10000 用户）
✅ **PostgreSQL**（首选）或 **MySQL**
- 高性能、高可用
- 支持主从复制、读写分离
- 适合大型网站、SaaS 平台

## 📦 迁移步骤

### 方式一：使用迁移工具（推荐）⭐

我为你准备了自动迁移脚本，支持一键迁移。

#### 1. PostgreSQL 迁移

```bash
# 1. 安装 PostgreSQL 驱动
pip install psycopg2-binary

# 2. 创建 PostgreSQL 数据库
sudo -u postgres psql
CREATE DATABASE planning_db;
CREATE USER planning_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE planning_db TO planning_user;
\q

# 3. 运行迁移脚本
python migrate_database.py \
  --from sqlite:///data/app.db \
  --to postgresql://planning_user:your_password@localhost/planning_db

# 4. 更新配置文件
# 编辑 data/config.yaml
database_path: postgresql://planning_user:your_password@localhost/planning_db

# 5. 启动应用
python app.py
```

#### 2. MySQL/MariaDB 迁移

```bash
# 1. 安装 MySQL 驱动
pip install pymysql

# 2. 创建 MySQL 数据库
mysql -u root -p
CREATE DATABASE planning_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'planning_user'@'localhost' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON planning_db.* TO 'planning_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;

# 3. 运行迁移脚本
python migrate_database.py \
  --from sqlite:///data/app.db \
  --to mysql+pymysql://planning_user:your_password@localhost/planning_db

# 4. 更新配置文件
# 编辑 data/config.yaml
database_path: mysql+pymysql://planning_user:your_password@localhost/planning_db

# 5. 启动应用
python app.py
```

### 方式二：手动迁移

如果不想使用自动脚本，可以手动导出导入。

#### 使用 SQLAlchemy 导出导入

```python
# export_data.py
from app import create_app
from models import db, User, Comment, BackupConfig  # 导入所有模型

app, _ = create_app()

with app.app_context():
    # 导出所有数据为 JSON
    import json
    
    data = {
        'users': [
            {
                'id': u.id,
                'username': u.username,
                'email': u.email,
                # ... 其他字段
            }
            for u in User.query.all()
        ],
        'comments': [
            {
                'id': c.id,
                'content': c.content,
                # ... 其他字段
            }
            for c in Comment.query.all()
        ],
        # ... 其他表
    }
    
    with open('data_export.json', 'w', encoding='utf-8') as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    
    print("数据导出完成！")
```

```python
# import_data.py
from app import create_app
from models import db, User, Comment, BackupConfig

app, _ = create_app()

with app.app_context():
    import json
    
    with open('data_export.json', 'r', encoding='utf-8') as f:
        data = json.load(f)
    
    # 创建表结构
    db.create_all()
    
    # 导入用户
    for user_data in data['users']:
        user = User(**user_data)
        db.session.add(user)
    
    # 导入评论
    for comment_data in data['comments']:
        comment = Comment(**comment_data)
        db.session.add(comment)
    
    # ... 导入其他表
    
    db.session.commit()
    print("数据导入完成！")
```

## 🔧 配置说明

### 数据库连接字符串格式

在 `data/config.yaml` 中配置 `database_path`：

```yaml
# SQLite（默认）
database_path: sqlite:///data/app.db

# PostgreSQL
database_path: postgresql://username:password@host:port/database
# 示例
database_path: postgresql://planning:secret@localhost:5432/planning_db

# MySQL/MariaDB
database_path: mysql+pymysql://username:password@host:port/database
# 示例
database_path: mysql+pymysql://planning:secret@localhost:3306/planning_db

# SQL Server
database_path: mssql+pyodbc://username:password@host/database?driver=ODBC+Driver+17+for+SQL+Server

# Oracle
database_path: oracle+cx_oracle://username:password@host:port/?service_name=service
```

### 连接池配置（可选）

对于生产环境，建议配置连接池：

```python
# 在 app.py 的 create_app() 中添加
app.config['SQLALCHEMY_ENGINE_OPTIONS'] = {
    'pool_size': 10,          # 连接池大小
    'pool_recycle': 3600,     # 连接回收时间（秒）
    'pool_pre_ping': True,    # 连接前检查
    'max_overflow': 20,       # 最大溢出连接数
}
```

## ⚠️ 注意事项

### SQLite 特有功能

如果你的代码使用了 SQLite 特有的功能，迁移时需要注意：

1. **日期时间处理**：
   ```python
   # SQLite
   created_at = db.Column(db.DateTime, default=datetime.utcnow)
   
   # PostgreSQL/MySQL（推荐）
   from sqlalchemy.sql import func
   created_at = db.Column(db.DateTime, server_default=func.now())
   ```

2. **布尔类型**：
   ```python
   # SQLite 存储为 0/1
   # PostgreSQL/MySQL 有原生 BOOLEAN 类型
   # SQLAlchemy 会自动处理，无需修改代码
   ```

3. **自增主键**：
   ```python
   # 所有数据库都支持，无需修改
   id = db.Column(db.Integer, primary_key=True)
   ```

### 字符编码

确保使用 UTF-8 编码：

```python
# PostgreSQL
database_path: postgresql://user:pass@host/db?client_encoding=utf8

# MySQL
database_path: mysql+pymysql://user:pass@host/db?charset=utf8mb4
```

### 事务隔离级别

不同数据库的默认隔离级别不同：

```python
# 如需指定隔离级别
app.config['SQLALCHEMY_ENGINE_OPTIONS'] = {
    'isolation_level': 'READ COMMITTED'  # 或 REPEATABLE READ
}
```

## 🔄 回滚方案

如果迁移后发现问题，可以快速回滚：

```bash
# 1. 停止应用
systemctl stop your-app

# 2. 恢复配置
cp data/config.yaml.backup data/config.yaml

# 3. 恢复数据库（如果是 SQLite）
cp data/app.db.backup data/app.db

# 4. 重启应用
systemctl start your-app
```

## 💡 常见问题

### Q: 迁移会丢失数据吗？

**A**: 使用推荐的迁移工具不会丢失数据。但务必先备份！

### Q: 迁移需要多长时间？

**A**: 取决于数据量：
- 小型数据库（< 100MB）：几分钟
- 中型数据库（100MB - 1GB）：10-30分钟
- 大型数据库（> 1GB）：可能需要几小时

### Q: 可以在线迁移吗？

**A**: 不建议。建议：
1. 维护窗口期间迁移
2. 或使用主从复制实现零停机迁移

### Q: 迁移后性能会提升多少？

**A**: 
- 读密集型应用：3-5倍提升
- 写密集型应用：10-20倍提升
- 高并发场景：显著提升

## 总结

从 SQLite 迁移到 PostgreSQL 或 MySQL 可以显著提升性能和并发能力。使用本指南提供的工具和步骤，可以安全、快速地完成迁移。
