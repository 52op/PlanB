# 用户管理系统说明

## 功能概述

新的用户管理系统提供了完整的用户和评论管理功能，独立页面 `/admin/users` 便于集中管理。

## 主要功能

### 1. 用户列表管理

访问路径：`/admin/users`

功能特性：
- 用户列表展示（ID、用户名、邮箱、角色、验证状态、评论权限、评论数）
- 多维度筛选：
  - 关键词搜索（用户名/邮箱）
  - 角色筛选（admin/regular/guest）
  - 邮箱验证状态筛选
  - 评论权限筛选
- 分页显示（每页 20 条）
- 快速跳转到用户详情页

### 2. 用户详情页

访问路径：`/admin/users/{user_id}`

包含三个标签页：

#### 基本信息
- 显示用户头像、用户名、昵称、邮箱、角色等
- 编辑用户信息：
  - 修改用户名
  - 修改昵称
  - 修改邮箱（修改后自动标记为未验证）
  - 修改角色（admin 账号除外）
  - 重置密码
- 标记邮箱已验证
- 删除用户（admin 账号除外）

#### 评论权限
- 全局评论权限开关
- 一键禁用/启用用户评论功能
- 禁用后用户无法在任何文章下发表评论

#### 用户评论
- 显示该用户的所有评论
- 按状态筛选（全部/待审核/已通过/已删除）
- 评论操作：
  - 通过审核
  - 删除评论
  - 查看文章
- 显示评论时间、文章名称、内容

### 3. 新增用户

在用户列表页面可以快速新增用户：
- 用户名（必填）
- 邮箱（可选）
- 密码（必填）
- 角色选择（guest/regular/admin）

## 数据库变更

### User 表新增字段

```sql
ALTER TABLE users ADD COLUMN can_comment BOOLEAN NOT NULL DEFAULT 1;
```

- `can_comment`: 是否允许评论（默认 true）

## 权限控制

### 评论权限检查

用户发表评论时会进行以下检查（按顺序）：

1. 是否已登录
2. 邮箱是否已验证
3. 是否有评论权限（`can_comment` 字段）
4. 评论内容是否为空

任何一项不满足都会抛出相应的错误提示。

### 管理员权限

- 只有 `role='admin'` 的用户可以访问用户管理页面
- admin 账号的角色不可修改
- admin 账号不可删除

## 路由列表

| 路由 | 方法 | 功能 |
|------|------|------|
| `/admin/users` | GET | 用户列表页 |
| `/admin/users/add` | POST | 新增用户 |
| `/admin/users/{user_id}` | GET | 用户详情页 |
| `/admin/users/{user_id}/update` | POST | 更新用户信息 |
| `/admin/users/{user_id}/verify-email` | POST | 标记邮箱已验证 |
| `/admin/users/{user_id}/delete` | POST | 删除用户 |
| `/admin/users/{user_id}/toggle-comment` | POST | 切换评论权限 |
| `/admin/comments/{comment_id}/approve` | POST | 通过评论 |
| `/admin/comments/{comment_id}/delete` | POST | 删除评论 |

## 使用场景

### 场景 1：禁止某用户评论

1. 进入 `/admin/users`
2. 搜索或筛选找到目标用户
3. 点击用户名进入详情页
4. 切换到"评论权限"标签
5. 取消勾选"允许该用户发表评论"
6. 系统自动保存

该用户再次尝试评论时会看到提示："你的评论权限已被管理员禁用"

### 场景 2：批量审核用户评论

1. 进入 `/admin/users`
2. 点击目标用户进入详情页
3. 切换到"用户评论"标签
4. 点击"待审核"筛选按钮
5. 逐条审核：点击"通过"或"删除"

### 场景 3：修改用户邮箱

1. 进入用户详情页
2. 在"基本信息"标签中修改邮箱
3. 点击"保存修改"
4. 系统自动将邮箱验证状态改为"未验证"
5. 如需标记为已验证，点击"标记邮箱已验证"按钮

### 场景 4：查看用户评论统计

在用户列表页面可以直接看到每个用户的评论总数，点击用户名进入详情页可以查看具体评论内容和状态分布。

## 邮件通知

当管理员对评论进行操作时，系统会自动发送邮件通知：

- 评论通过审核：发送"评论审核通过"邮件
- 评论被删除：发送"评论未通过审核"邮件

前提条件：
- 用户邮箱已验证
- 系统已配置邮件服务器
- 评论状态发生变化（pending → approved 或 → deleted）

## 注意事项

1. **admin 账号保护**：
   - admin 账号的角色不可修改
   - admin 账号不可删除
   - 这是系统安全的基本保障

2. **邮箱验证状态**：
   - 修改邮箱后会自动标记为未验证
   - 未验证邮箱的用户无法发表评论
   - 管理员可以手动标记为已验证

3. **删除用户**：
   - 删除用户会同时删除其所有评论
   - 删除用户会同时删除其权限规则
   - 此操作不可恢复，请谨慎操作

4. **评论权限**：
   - 禁用评论权限后，用户无法发表新评论
   - 已发表的评论不会被删除
   - 可以随时重新启用

## 界面优化

- 使用 Badge 标签清晰显示状态
- 颜色编码：
  - 管理员：蓝色
  - 普通用户：紫色
  - 访客：橙色
  - 已验证：绿色
  - 未验证：红色
  - 待审核：黄色
  - 已通过：绿色
  - 已删除：红色

- Tab 切换流畅，无需刷新页面
- 评论筛选实时生效
- 响应式设计，适配移动端
