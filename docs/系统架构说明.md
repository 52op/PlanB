# 项目架构说明

本项目采用 Flask 框架构建，并遵循“应用工厂”模式，以实现模块化和可维护性。

## 目录结构

```
planning/
├── blueprints/           # 蓝图目录，存放各个模块的路由
│   ├── __init__.py
│   ├── admin.py          # 后台管理模块
│   ├── api.py            # API 接口模块
│   ├── auth.py           # 用户认证模块
│   └── main.py           # 核心文档展示模块
├── docs/                 # 项目文档（本文件所在目录）
├── instance/             # 实例文件夹，通常用于存放数据库文件等
├── jobs/                 # 默认的 Markdown 文档存放目录
├── static/               # 静态文件
│   └── uploads/          # 本地上传的媒体文件
├── templates/            # Jinja2 模板
├── venv/                 # Python 虚拟环境
├── app.py                # 应用入口和工厂函数
├── config.yaml           # 应用配置
├── models.py             # SQLAlchemy 数据模型
├── plan.md               # 原始开发计划
├── requirements.txt      # Python 依赖
└── services.py           # 核心业务逻辑服务
```

## 核心模块

### `app.py`

作为应用的入口，包含 `create_app()` 工厂函数。该函数负责：
- 创建 Flask 应用实例。
- 从 `config.yaml` 加载配置。
- 初始化数据库 (`Flask-SQLAlchemy`) 和登录管理器 (`Flask-Login`)。
- 注册 `blueprints` 目录下的所有蓝图。

### `blueprints/`

此目录下的每个文件都定义了一个独立的 **蓝图 (Blueprint)**，将应用按功能划分为不同模块：
- **`main`**: 处理核心的文档查看、首页展示、目录浏览以及本地媒体文件访问。
- **`admin`**: 负责所有 `/admin` 前缀下的后台管理功能，如用户管理、系统设置、权限控制等。
- **`api`**: 提供所有 `/api` 前缀的接口，如内容保存、文件上传（包括 Markdown 和媒体文件）。
- **`auth`**: 处理用户登录、登出等认证相关的功能。

### `services/`

服务层目录，封装了应用的核心业务逻辑，按功能模块划分：

- **文档服务** (`docs.py`)：Markdown 文件解析、渲染、文件树构建
- **备份系统**：
  - `backup_engine.py`：备份核心引擎
  - `backup_scheduler.py`：定时任务调度
  - `backup_validator.py`：备份文件验证
  - `backup_restorer.py`：备份恢复功能
  - `backup_notification.py`：备份通知服务
  - `backup_config.py`：备份配置管理
  - `backup_storage_monitor.py`：存储监控
- **存储适配器** (`storage/`)：
  - `ftp_adapter.py`：FTP 存储
  - `s3_adapter.py`：S3 兼容存储
  - `email_adapter.py`：邮件存储
- **其他服务**：
  - 权限检查、评论管理、分享链接、邮件发送等

### `models.py`

定义了应用所需的所有数据模型，使用 `Flask-SQLAlchemy` 进行 ORM 映射。

主要数据模型：
- **用户系统**：User, PermissionRule, PasswordAccessRule
- **内容系统**：Comment, DocumentViewStat, ShareLink
- **备份系统**：BackupConfig, BackupJob, BackupFileTracker
- **系统配置**：SystemSetting, DirectoryConfig
- **其他**：Image, EmailVerificationCode, RateLimitAttempt, CoverFallbackCache, NotificationLog

### 数据库管理

- **自动迁移**：`db_sync.py` - 启动时使用 `--update-db` 参数自动对比 models.py 和数据库结构
- **数据库切换**：`migrate_database.py` - 支持从 SQLite 迁移到 PostgreSQL、MySQL 等

## 请求流程示例（查看文档）

1.  用户访问 `http://host/doc/some/document.md`。
2.  请求被 `main` 蓝图中的 `doc()` 路由捕获。
3.  `doc()` 函数调用 `services.py` 中的 `read_markdown_file()`。
4.  `read_markdown_file()` 从文件系统读取 `.md` 文件，使用 `markdown` 库将其渲染为 HTML。
5.  最终，`doc()` 函数使用 `render_template()` 将渲染后的 HTML 和其他页面数据（如文件树）填充到 `index.html` 模板中，并返回给用户。
